설치

  • 소프트웨어 저장

    자동 인스톨 패키지가 아니므로 미리 폴더 하나를 생성해 둡니다.

    생성한 폴더에 아이탐 소프트웨어를 저장하세요.

  • 소프트웨어 실행

    위에서 생성한 폴더 위치로 이동하고(중요!) 소프트웨어를 실행합니다. 실행된 소프트웨어를 "에이전트" 라 부릅니다.

    • windows

      명령프롬프트에서 아래와 같이 실행합니다.

      java -jar ITAhM.jar

      또는 ITAhM.exe 실행파일을 실행시킵니다.

      컴퓨터 재시작시 자동으로 실행되도록 시작프로그램으로 등록합니다.

    • linux

      콘솔에서는 아래와 같이 실행합니다.

      sudo java -jar ITAhM.jar &

      터미널에서는 아래와 같이 실행합니다.

      nohup java -jar ITAhM.jar &

      컴퓨터 재시작시 자동으로 실행되도록 upstart 또는 init.d 에 등록합니다.

시작

  • 에이전트 접속

    브라우저에서 http://itahm.com/connect.html 페이지를 엽니다.

    에이전트의 주소1 를 입력합니다.

    기본 TCP 는 2014입니다. 변경하여 실행한 경우 변경한 TCP를 입력하세요.2

    최초 기본 계정은 아이디3 root 비밀번호4 root 입니다.

  • 구성도

    장비의 추가, 변경, 연결구성 등은 "구성도" 메뉴에서 수행합니다. 구성도 좌측 상단에 설정 아이콘1을 누르면 추가2, 그룹3, 저장4 버튼이 활성화 됩니다.

    구성도의 수정중에는 이 상태를 유지하십시오. 그렇지 않으면 이벤트 수신시 구성도가 새로고쳐지기 때문에 수정 전의 상태로 초기화 됩니다.

    추가버튼2 을 클릭해 장비를 추가합니다.

    그룹버튼3 을 클릭해 그룹을 추가합니다.

    장비가 등록되면 중앙을 기준으로 (0, 0)좌표에 그려집니다. 이동시키지 않으면 모두 같은 위치에 그려지므로 등록후 적당한 위치로 이동시키십시오.

    장비를 이동시키려면 먼저 장비를 클릭하여 선택을 완료한 후 다시 클릭하여 드래그 합니다. Shift 키를 누른 상태로 선택하면 여러 장비를 선택하여 이동시킬 수 있습니다.

    장비가 그룹간 이동하는 경우 연결을 재설정 하기 위하여 링크(연결선)는 모두 제거 됩니다.

    장비를 선택하지 않고 드래그 하는 경우 구성도 전체가 이동합니다.

    장비의 위치와 링크의 변경사항은 자동으로 저장되지 않으므로 반드시 관리자 권한이 있는 계정으로 저장버튼4 을 클릭하여 저장하십시오.

    장비 또는 그룹의 삭제시에는 연관정보의 갱신을 위해 자동으로 저장됩니다.

  • 장비의 등록

    IP주소1 를 입력하세요. 호스트명이나 도메인명은 사용할 수 없습니다.

    장비의 이름2 은 생략할 수 있습니다.

    장비의 타입3 은 구성도에 그려지는 장비의 아이콘 모양을 결정합니다.

    장비 라벨4 은 둘 이상 복수개로 등록할 수 있습니다. 콤마(,)로 구분하여 주십시오.

    장비의 정보를 추가적으로 관리하고자 하는 경우 설정 커스텀 항목5을 생성할 수 있습니다. 설정 매뉴얼을 참고하십시오.

    장비 또는 그룹의 삭제시에는 연관정보의 갱신을 위해 자동으로 저장됩니다.

  • SNMP 프로파일

    아이탐은 SNMP 모니터링 대상 장비들의 개별 SNMP 정보를 가지지 않고 미리 프로파일에 등록되어 있는 공통정보를 통해 SNMP 요청을 시도합니다.

    장비의 모니터링을 활성화 하기 이전에 반드시 프로파일이 설정되어 있어야 합니다. 기본적으로 default1 라는 이름의 프로파일이 생성됩니다. 이 프로파일은 포트2 UDP 161 버전3 v2c 커뮤니티4 public 의 설정정보를 가지고 있습니다.

    기본 프로파일을 수정하거나 모니터링 대상장비들이 사용중인 모든 프로파일을 등록하십시오. 등록된 프로파일을 사용하는 장비가 존재한다면 이 프로파일은 삭제할 수 없습니다.4

  • 모니터링

    장비를 등록하고 나면 모니터링1 방식을 선택할 수 있습니다. SNMP2 와 ICMP3 프로토콜중 한가지를 선택하십시오. SNMP를 통해 모니터링 한다고 해도 Health 체크는 ICMP를 사용하므로 장비는 모니터링 방식에 관계 없이 에이전트로부터 ICMP를 허용해야 합니다.

    자동탐색6 기능을 사용하여 네트워크 대역별로 SNMP가 활성화된 장비를 탐지할 수 있습니다. ICMP는 탐지하지 않습니다. (불필요한 장비의 탐지 방지)

사용

  • 목록

    좌측 상단a 에 표시된 숫자가 등록된 장비의 합계 입니다.

    장비의 상태별로 필터링b 하여 목록에 보여줄 수 있습니다. 장애, 임계, 정상, 비활성, 모두보기의 총 5가지 옵션이 있습니다.

    장비의 그룹(라벨)c 별로 구분하여 목록에 보여줄 수 있습니다. 장비의 정보에 설정한 라벨별로 볼수 있고 라벨이 없는 장비와 모든 장비를 볼 수 있는 추가 옵션이 있습니다.

    장비의 목록은 저장버튼d 을 클릭하여 csv(comma separated values) 파일로 저장할 수 있습니다. 파일은 UTF-8로 인코딩은 되는데 마이크로소프트 엑셀에서 열람할때는 ANSI로 인식하기 때문에 한글이 제대로 표시되지 않습니다. 메모장이나 다른 텍스트 에디터로 편집하여 ANSI로 변환하여 이 문제를 해결하십시오. 또한 csv 파일은 comma(,)를 구분자로 하는 파일이므로 장비의 정보에 comma가 들어있다면 원하는 정보로 가공할 수 없습니다.

    목록에서 장비를 선택하는 경우 버튼들e, f, g, h, i 이 나타나게 되는데 비활성 장비이거나 ICMP 모니터링 대상장비인 경우에는 버튼들e, f 만 보여줍니다.

    버튼e 를 클릭하면 선택된 장비의 위치를 찾아서 그 장비를 중심으로 하는 구성도로 이동합니다.

    버튼f 를 클릭하면 선택된 장비의 등록정보 대화상자를 보여줍니다.

    버튼g 를 클릭하면 선택된 장비의 성능 정보로 이동합니다.

    버튼i 를 클릭하면 선택된 장비의 임계 설정 대화상자를 보여줍니다.

    키워드j 를 기준으로 목록을 필터링할 수 있습니다. 목록이 이미 라벨별로 구분되어 있는 경우 구분을 유지하면서 필터링 합니다.

    키워드 필터링을 해제하고자 하는 경우 초기화 버튼k 을 클릭합니다.

  • 성능
    구성도 또는 목록에서 장비를 선택하하면 컨트롤이 활성화 되고 버튼4 을 클릭하여 성능 화면으로 이동할 수 있습니다.

    장비의 기본 snmp 정보1 를 보여줍니다.

    서버의 경우 실행상태에 있는 프로세스의 갯수2 를 보여줍니다.

    프로세서의 평균 로드3, 물리적 메모리 사용량4, 응답속도5, 스토리지 사용량6, 통신 사용량7 및 에러c 등을 게이지 형식으로 보여주며 각 게이지를 클릭하여 기간별 차트를 검색할 수 있습니다.

    통신의 경우 비활성 인터페이스를 감추거나 보이게b 할 수 있고 연결되지 않은 활성 인터페이스는 빨간 원a 으로 표시됩니다.

    모든 게이지는 70% 이하 초록색, 90%이하 주황색, 90%초과시 빨간색으로 표시됩니다.

  • 차트

    최초 일주일간의 데이터를 불러오고, 하루(오늘)의 차트를 보여줍니다.

    불러온 데이터의 시작1 과 끝2 사이에 있는 바를 좌(이전)우(이후)로 이동시키거나 바의 왼변3 을 좌(이전)우(이후)로 드래그하여 차트의 기간을 변경합니다.

    차트에 마우스를 올리면 마우스 위치의 시간5 과 평균값6, 최대값7, 최소값8을 표시해 줍니다.

    차트의 색칠된 영역은 최대값과 최소값에 의해 결정되고 내부의 굵은 흰색 실선은 평균값을 의미합니다. 하루간(24시간)의 데이터를 그릴때는 1분 또는 5분(설정에 의함)단위의 세부 차트9 를 막대그래프로 함께 표시해 줍니다.

    검색할 차트의 기간은 제한 없이 재설정a 가능합니다.

    선택한 자원의 실시간 차트b 를 5초간격으로 갱신하여 연속적으로 보여줍니다.

    차트의 데이터를 csv 파일로 저장c 할 수 있습니다.

    차트를 인쇄d 할때 용지의 크기에 맞게 차트를 조정합니다.

  • 임계
    구성도 또는 목록에서 장비를 선택하하면 컨트롤이 활성화 되고 버튼3 을 클릭하여 임계 설정 대화상자를 엽니다.

    각 장비의 성능지표1 마다 임계값3 을 설정하여 이벤트를 발생 시킬수 있습니다.

    스토리지와 인터페이스와 같이 장비마다 여러개가 있는 지표는 구분2 하여 임계값을 설정 합니다.

    멀티코어 프로세서의 경우 코어당 임계값을 설정하지 않고 임계값을 넘어서는 코어가 존재할때 이벤트를 알립니다.

  • 연결
    구성도에서 장비간 연결선을 추가할때는 장비를 클릭하여 선택을 완료한 후 버튼1 을 드래그하여 연결 대상 장비에 드랍하십시오.

    연결 가능한 상태까지 드래그가 되면 붉은색이던 연결 점선이 초록색으로 변경됩니다.

    드랍 후 연결 대화상자가 열리면 각 장비의 연결에 해당하는 인터페이스를 선택하십시오. SNMP가 활성화 되지 않았거나 ICMP 모니터링 대상 장비의 인터페이스 선택 컨트롤은 선택할수 없는 상태가 됩니다.

  • 이벤트

    날짜별로 발생한 이벤트를 검색할 수 있습니다.

    장애가 발생한 경우3 와 임계를 넘어선 경우2 그리고 장애나 임계 상황에서 벗어난 경우1 세가지 상황에 따라 아이콘을 구분하여 표시합니다.

  • 스크랩

    특정 장비의 차트가 아닌 여러 장비의 여러 차트를 비교하거나 리포트하기 위하여 스크랩 기능을 활용합니다.

    스크랩을 원하는 날짜의 시작1 과 끝2 을 결정합니다. 시작은 포함하고 끝은 포함하지 않습니다. 예를 들어 오늘이 20일이고 오늘 하루의 차트를 스크랩하는 경우 20일부터 21일 까지 입니다.

    최초 1열이 제공되는데 열을 추가하고자 하는 경우 열추가 버튼3 을 클릭합니다.

    해당 열에 차트를 추가하고자 하는 경우 버튼4 을 클릭합니다.

    해당 열을 삭제하고자 하는 경우 버튼5 을 클릭합니다.

    차트 추가 버튼을 눌러 스크랩 대화상자를 연후 장비의 IP를 입력하고 확인버튼을 클릭합니다.

    해당 장비의 성능지표가 나열되면 선택3 하여 스크랩2 을 진행합니다. 이전 단계로 되돌려 다른 장비를 선택하고자 하는 경우 취소1 할 수 있습니다.

    열 하나를 추가하여 두개의 열을 만들고 좌측열에는 응답시간과 프로세서 로드를, 우측 열에는 스토리와 인터페이스 사용량을 스크랩한 결과물 입니다.

    다른 기간을 스크랩하고자 하는 경우 먼저 기간을 설정하고 스크랩 하면 바뀐 기간으로 차트를 불러옵니다.

    각 차트들은 같은 열 내 상하로 드래그할 수 있고 다른 열간 좌우로 드래그하여 레이아웃을 변경할 수 있습니다.

설정

  • 정보

    에이전트가 설치된 서버의 자바 버전을 표시합니다. 안정된 버전(8 이상) 또는 최신의 자바 버전을 유지하십시오.

    현재 서비스 하고 있는 에이전트의 버전을 표시합니다.

    에이전트가 설치된 서버의 저장공간 여유분과 전일 사용된 디스크 용량을 참고하여 데이터 보관 기간을 설정하십시오. 저장 여유공간 / 전일 디스크 사용량 = 보관 가능한 데이터 보관 일수

    데이터베이스 로드는 에이전트가 수집한 정보를 데이터베이스에 저장하는데 소요되는 시간을 의미합니다. 단위는 ms(milliseconds)이고 이 값은 1000ms 이내로 유지되어야 안정적이므로 넘어서는 경우 서버의 업그레이드를 고려하여야 할 중요한 지표입니다.

    각 장비가 제공하는 프로세서, 메모리, 스토리지, 인터페이스 정보를 모두 합한 값이 관리 리소스 갯수 입니다.

    계약 형태에 따라 에이전트 사용기간의 만료일이 정해집니다. 무제한인 경우 N/A(Not Applicable) 로 표시됩니다.

  • 일반

    데이터 보관 기간을 일 단위로 설정할 수 있습니다. 0으로 설정시 무제한 입니다.

    데이터의 저장 간격을 1분 또는 5분으로 지정할 수 있습니다. 기본 1분이며 저장되는 데이터가 저장공간에 비해 너무 많다면 5분으로 변경할 수 있습니다.

    대쉬보드나 구성도에 장비를 표시할때 장비의 이름을 보여주거나 IP를 보여주도록 선택할 수 있습니다.

    대쉬보드가 Top10 목록을 보여주기에 너무 작다면 목록의 갯수를 조정하십시오. 기본 10개(최대) 이며 이보다 작은 수로 변경할 수 있습니다.

    상단 메뉴를 고정하거나 자동으로 작아지도록 설정할 수 있습니다. 공간이 충분하다면 메뉴를 고정할 수도 있고 메뉴를 자동으로 축소하도록 하여 대쉬보드나 구성도를 더 간결하게 보여줄 수도 있습니다.

    기업이 장비를 관리할때 사용하는 특별한 정보가 있다면 추가할 수 있습니다. ,(comma) 로 구분하여 1개 이상의 정보를 입력 하십시오.

    만일을 대비하여 설정정보를 백업하십시오. 또는 같은 설정을 갖는 또다른 서버를 구축하기 위하여 백업기능을 사용할 수 있습니다.

  • 대쉬보드

    대쉬보드에 표시할 성능정보의 내용을 변경할 수 있습니다.

    선택을 해제1하면 감추고 푸른색 화살표 버튼2을 클릭하면 순서가 바뀝니다. 대쉬보드에는 감추어 지거나 바뀐 순서로 표시됩니다.

  • 계정

    최초 설치시 기본 계정은 아이디1 root, 비밀번호2 root 이며 관리자 권한입니다. 보안을 위하여 관리자 권한이 없는 사용자4 계정을 추가하세요.

    계정을 추가하기 위하여 추가 버튼을 클릭하여 계정 대화상자를 엽니다.

    비밀번호는 모든 특수문자의 사용이 가능하며 6자 이상이어야 합니다.

    관리자 계정은 추가/변경/삭제/저장 등의 권한이 있으며 사용자 계정은 읽기 권한만을 가집니다.

    생성되어 있는 모든 계정을 삭제하고 서비스를 재시작 하면 최초 계정 상태로 초기화 됩니다.

소프트웨어는 자바 언어로 작성되었고 이를 실행시키기 위하여 컴퓨터에 자바가 설치되어 있어야 합니다. 자바 버전은 최신의 상태로 유지하여 주십시오.

관리자에 의해 수정된 정보가 존재한다면 재시작 메뉴를을 클릭하여 정보를 새로고침 하십시오. 관리자가 구성도의 수정(장비의 위치와 연결)사항을 저장하지 않고 원래 상태로 되돌리고자 하는 경우에도 재시작 하십시오.

구성도에서 사용되는 장비의 아이콘 모양을 회사의 이미지에 맞게 변경할 수 있습니다.
